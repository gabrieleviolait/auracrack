# Security policy

## Supported version

Only the latest version of Aura Security Suite on the `main` branch receives security fixes. The standalone `auracrack*.py` scripts are retained for historical compatibility and are not the recommended interface.

## Reporting a vulnerability

Please do not open a public issue for vulnerabilities that could expose credentials, personal data or enable abuse. Report them privately through GitHub Security Advisories using the repository's **Security → Report a vulnerability** function.

Include the affected file/version, reproduction steps, expected impact and any suggested mitigation. Do not include real credentials, packet captures or data belonging to third parties. A report should be acknowledged within 7 days when reasonably possible.

## Scope and safe testing

Test only with synthetic data and on systems and networks you own or are explicitly authorized to assess. Do not perform denial-of-service testing, collect third-party traffic or publish working secrets as part of a report.
